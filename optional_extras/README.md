# Optional extras

This page is a roadmap, not a claim about what was required or used in the preparation documented by this repository.

The core material was sufficient for the process it describes. Someone interviewing for a different mix of product-data-science, research, applied-ML, or software-engineering roles may choose to add selected topics below.

## General algorithms and data structures

Study these only when the target loop explicitly emphasizes general coding:

- stacks and queues;
- linked lists;
- trees and binary-search trees;
- breadth-first and depth-first search;
- graph representations and traversal;
- union-find;
- tries;
- backtracking;
- common dynamic-programming patterns.

Binary search is in the core folder because it was part of the actual preparation.

Possible extensions for experimentation-specialist roles:

- switchback experiments;

- regression discontinuity;

## How to choose

Do not study everything by default.

1. Identify the actual interview formats.
2. Add only subjects clearly relevant to those formats.
3. Practice representative problems rather than aiming for exhaustive coverage.
4. At senior and staff levels, reserve substantial time for communication and project stories.
